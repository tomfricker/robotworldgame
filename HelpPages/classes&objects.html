Level 2 looks at Objects and Classes. This is a crucial concept to any Object-Oriented Programming (OOP) language including Java which is what we teach in RobotWorld.
Here is the Theory we go through, so that you can access it at any time!

Introduction:

In the level we initially get introduced to two classes: the Robot and the Item Classes. Robot is an object from the Robot Class. This means it has characteristics (fields) that are taken from (belong to) the Robot Class.

You should note that Classes start with a capital letter, while objects are written with a small letter. Classes have methods that help you make an action, but only an object can invoke that method.

The apple and the banana in this Level are Objects as well, instances from the Class Item. The robot can perform actions on them with the method eat(). You can invoke the method by writing code in the form:

ObjectName.MethodName(parameter);

So practically putting that in use with our Objects, it would look like this:

robot.eat(apple);

Here the Method eat() takes a parameter, that is an object of Class Item. You will get a better understanding of this further down when looking at methods in more detail.

Please note that you can name objects as you wish, as long as they start with a small letter and do not have a pre-existing meaning in Java (so we would exclude words like String, boolean, public, private etc.). The same rules apply when naming a Class, however the Class name needs to start with a capital letter.

To create an object from a Class you need to write code in the following form:

ClassName ObjectName = new ClassName();

The second part of this expression (after the = sign) is called a Constructor. This is another element of a Class just like Fields and Methods.

As an example, let us create some Objects from our existing Classes:

Item apple = new Item();

Robot robot = new Robot();

The Class Structure

As we have seen above a Class in Java has 3 main components: Fields (that refer to Objects' state), Methods (that refer to Objects' behaviour) and Constructor (that is responsible for Objects' creation).

When building a class, you would follow the bellow structure:


public class ClassName

{

	Fields

	Constructor

	Methods

}

A full class example will be given at the end.

Access Modifiers

Before we go into more detail about each of these components, it is important to look at one last concept: access modifiers.

These are key words that, just as their name suggests, modify the access of Classes and their elements to other Classes and their elements. There are 4 types of access modifiers - private, default, protected and public; but we will only concentrate on 2 of those - public and private.

The private modifier is only accessible within the Class, while the public one is accessible everywhere. We normally put access modifiers when defining a Class, Constructor, fields or methods.

Classes, Constructors and methods usually have public access modifier, so that they can be used in other Classes as well.

Fields normally have a private access modifier so that their values cannot be changed outside of the Class they are in.

Examples of access modifiers and how they are used:

Class


public class Item 

{ 

	Class Body 

}

Constructor


public Item() { 

	Constructor Body

}

Method


public String getName(){

	Method Body

}

Field


private int age;

Introduction to Fields

As mentioned before fields represent the characteristics of a Class and the objects from that Class and allow for values to be stored within objects.

Each field has a Data Type that can be primitive or reference type: primitive as we have seen in Level: Data Types such as String, int or boolean and reference type that can be any Class such as Robot or Item.

We write Fields in Java like so:

accessModifier DataType/ClassName nameofField;

When you substitute all of these with concrete names you can get the following:

private Item apple;(this Field is of Class Item and is called apple)

private String address;(this Field is of type String, called address)

private boolean hasAddress;(this Field is of type Boolean and be either True or False value)

private String name;

private int age;

Introduction to Constructor

Constructors are used when a new object of a Class is created. They have the same name as the Class and initialise the object by storing initial values for the fields. These values can be passed by parameters or hard coded in the constructor's body.

Here are examples of both:


public Robot(String fullName, int robotAge) {

	//this is a comment in Java: here we equate the Fields we created earlier to the parameters above

	name = fullName;

	age = robotAge;

}



public Robot() {

	name = "Robbie";

	age = 15;

}

Every object created with the second constructor will have the name "Robbie" and the age of 15 as default. The objects created with the first constructor can have different name and age depending on what values you pass as parameters for fullName and robotAge.

Examples of creating an object with both constructors:

Robot robot = new Robot();

Robot robotOne = new Robot("Harry", 18);

As you can see, a Class can have multiple constructors, if they differ in parameters.

Introduction to Methods

Methods specify an action that is performed BY or ON an object. Each Class has certain methods, which its objects can perform or invoke.

Here is an example of how a method is defined within a Class:


accessModifier returnType methodName (parameters) {

	action;

	action;

	action;

}

Methods that return an object have the same return Type as object's Data Type (e.g. String, int, Robot) and are called accessor methods. They will also always have the return key word in their body:


public String getAddress() {

	return address;

}

Methods that change the state of an object and do not have a return statement have a VOID return Type and are called mutator methods.


public void changeAddress(String newAddress) {

	address = newAddress;

}

/**

 * This is the Robot class and it determines what Objects from the Class

 * have as characteristics and what they can do.

 * (You can write comments here and they won't be interpreted as code by the

 *

 */

public class Robot

{

    //the age of the robot

    private int age;

    //the name of the robot

    private String name;

    //the hat of the robot

    private Item hat;

    //if the robot has a buddy

    private boolean hasBuddy;

    //if the robot is hungry

    private boolean isHungry;



    /**

     * Constructor for objects of class Robot with parameters

     */

    public Robot(String fullName, int robotAge)

    {

        // initialise the fields

        name = fullName;

        age = robotAge;

        hat = new Item();

        hasBuddy = true;

        isHungry = true;

    }



    /**

     * Constructor for objects of class Robot without parameters

     */

    public Robot()

    {

        // initialise the fields

        name = "Robbie";

        age = 15;

        hat = new Item();

        hasBuddy = true;

        isHungry = true;

    }



    /**

     * Returns the age of the robot

     *

     * @return    the age of the robot

     */

    public int getAge()

    {

        return age;

    }

    

    /**

     * Sets the age of the robot to the parameter value

     *

     * @param  newAge  the age you're setting for the robot

     */

    public void setAge(int newAge)

    {

        age = newAge;

    }

    

    /**

     * Returns the name of the robot

     *

     * @return    the name of the robot

     */

    public String getName()

    {

        return name;

    }

    

    /**

     * Changes the name of the robot to the parameter value
 
     *

     * @param  newName  the new name of the robot

     */

    public void changeName(String newName)

    {

        name = newName;

    }

    

    /**

     * Returns whether the robot has a buddy or not

     *

     * @return   the hasBuddy value of the robot

     */

    public boolean getBuddy()

    {

        return hasBuddy;

    }

    

    /**

     * Returns whether the robot is hungry or not

     *

     * @return   the isHungry value of the robot

     */

    public boolean getHungry()

    {

        return isHungry;

    }

    

    /**

     * Set the hat colour of the robot

     *

     * @param  colour  the colour you're setting for the hat

     */

    public void setHatColour(String colour)

    {

        hat.changeColour("blue");

    }

    

    /**

     * The robot eats

     *

     * @param  fruit  the Item that the robot eats

     */

    public void eat(Item fruit)

    {

        fruit.disappear();

        isHungry = false;

    }

    

}
